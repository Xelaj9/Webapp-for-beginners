เว็บไซต์คืออะไรกันแน่?

ก่อนเรียนรู้ ต้องเข้าใจภาพรวมก่อน

เว็บไซต์คือไฟล์ที่เก็บอยู่บนคอมพิวเตอร์เครื่องหนึ่ง (เรียกว่า Server) และเมื่อคุณพิมพ์ URL ในเบราว์เซอร์ เบราว์เซอร์จะไปขอไฟล์นั้นมาแสดงให้คุณดู

📺
เปรียบเหมือน: ร้านพิซซ่า

คุณ (เบราว์เซอร์) โทรสั่งพิซซ่า → ร้าน (Server) รับออเดอร์ → ทำพิซซ่า (ส่งไฟล์) → ส่งมาให้คุณ (แสดงหน้าเว็บ)

ทุกครั้งที่คุณคลิก link หรือพิมพ์ URL คุณกำลัง "โทรสั่งพิซซ่า" นั่นเอง

HTML

HyperText Markup Language — โครงกระดูกของเว็บ

HTML คือภาษาที่บอกเบราว์เซอร์ว่า "หน้าเว็บมีอะไรบ้าง" — มีหัวข้อ มีข้อความ มีรูป มีปุ่ม อยู่ที่ไหน

📋
เปรียบเหมือน: แบบแปลนบ้าน

แบบแปลนบอกว่า "ห้องนอนอยู่ซ้าย ห้องน้ำอยู่ขวา ประตูอยู่กลาง" แต่ยังไม่ได้บอกว่าสีผนังเป็นอะไร
HTML ก็เหมือนกัน — บอกแค่ว่า "มีหัวข้ออยู่บนสุด มีรูปตรงกลาง มีปุ่มด้านล่าง" เท่านั้น

HTML ทำงานอย่างไร — Tag คืออะไร?
🏷️
Tag = ป้ายชื่อที่ติดให้เนื้อหา

ลองนึกถึงกล่องพัสดุ เราติด ป้าย "เปราะบาง" เพื่อบอกว่ากล่องนี้พิเศษ
HTML Tag ก็เหมือนกัน — ติดป้าย <h1> บอกว่า "นี่คือหัวข้อใหญ่"
ติดป้าย <p> บอกว่า "นี่คือย่อหน้าข้อความ"
ทุก tag มีเปิด <xxx> และปิด </xxx> เหมือนวงเล็บ

โค้ด HTML
<h1>สวัสดีครับ!</h1> <p>ฉันชื่อสมชาย อายุ 25 ปี</p> <img src="cat.jpg"> <button>คลิกที่นี่</button> <a href="google.com"> ไปกูเกิล </a>

CSS

Cascading Style Sheets — ช่างแต่งหน้าของเว็บ

CSS บอกเบราว์เซอร์ว่า "แต่ละอย่างในเว็บหน้าตาเป็นยังไง" — สีอะไร ขนาดเท่าไร อยู่ตรงไหน มีเงาไหม

💄
เปรียบเหมือน: ช่างแต่งตัวให้นักแสดง

HTML สร้าง "นักแสดง" (ข้อความ, ปุ่ม, รูป) ออกมาก่อน
CSS คือ "ช่างแต่งหน้า" ที่มาแต่งให้ดูดี — ใส่ชุด, แต่งหน้า, ทำผม
ถ้าไม่มี CSS เว็บจะดูธรรมดาเหมือนกระดาษข้อความล้วน ไม่มีสีไม่มีดีไซน์

CSS ทำงานอย่างไร?
🎯
เลือก → แต่ง (Selector → Property)

CSS ทำงาน 2 ขั้น:
1. เลือกว่าจะแต่งอะไร เช่น "ปุ่มทุกปุ่ม" หรือ "หัวข้อใหญ่"
2. บอกว่าแต่งยังไง เช่น "ทำให้สีแดง" หรือ "ขนาดตัวอักษร 20px"

โค้ด CSS
/* แต่ง h1 ทุกอัน */ h1 { color: royalblue; font-size: 32px; text-align: center; } /* แต่งปุ่มทุกปุ่ม */ button { background: #e8622a; color: white; padding: 12px 24px; border-radius: 8px; border: none; } /* เปลี่ยนเมื่อ hover */ button:hover { background: #c0471a; }

JavaScript

ภาษาที่ทำให้เว็บ "มีชีวิต" และโต้ตอบได้

JavaScript คือสิ่งที่ทำให้เว็บ ทำอะไรได้ — เมื่อกดปุ่มแล้วมีอะไรเกิดขึ้น เมื่อพิมพ์แล้วข้อมูลเปลี่ยน เมื่อเลื่อนหน้าแล้วมี animation

🤖
เปรียบเหมือน: ระบบในบ้านอัจฉริยะ

ถ้าบ้านมีแค่ HTML+CSS — มันเหมือนรูปถ่ายบ้าน สวยงาม แต่ทำอะไรไม่ได้
JavaScript คือ ระบบ Smart Home — กดปุ่มไฟติด, เปิดประตูอัตโนมัติ, แอร์ปรับตัวเอง
ทำให้บ้าน (เว็บ) จาก "รูปถ่าย" กลายเป็น "ของจริงที่ใช้งานได้"

แนวคิดพื้นฐาน JavaScript
📦
Variable (ตัวแปร) — กล่องเก็บของ

เหมือนกล่องมีชื่อ เก็บข้อมูลได้ เช่น กล่องชื่อ "score" เก็บคะแนน 100 คะแนน ถ้าทำคะแนนได้เพิ่มก็แค่เปลี่ยนเลขในกล่อง ไม่ต้องสร้างกล่องใหม่

🔧
Function (ฟังก์ชัน) — สูตรอาหาร

เหมือนสูตรอาหารที่เขียนไว้ครั้งหนึ่ง แล้วทำซ้ำได้ตลอด เช่น สูตร "ทอดไข่" บอกขั้นตอนไว้ พอจะทอดไข่แค่ "เรียกสูตร" ไม่ต้องคิดใหม่ทุกครั้ง

🔍
If/Else — การตัดสินใจ

เหมือนสัญญาณไฟจราจร ถ้าไฟเขียว = ไปได้, ถ้าไฟแดง = หยุด JavaScript ก็ตัดสินใจแบบนี้ เช่น "ถ้าคะแนน ≥ 50 → ผ่าน ถ้าไม่ → ไม่ผ่าน"

🔄
Loop (ลูป) — การทำซ้ำ

เหมือนนับลูกแก้ว แทนที่จะนับทีละเม็ด สั่งว่า "วนซ้ำจนครบ 100 เม็ด" JavaScript ก็ทำซ้ำๆ ได้โดยไม่เหนื่อย

📋
Array (อาเรย์) — รายการ

เหมือนรายการซื้อของ เก็บหลายอย่างในที่เดียว เช่น ["นม", "ไข่", "ขนมปัง"] แทนที่จะสร้างตัวแปรแยกสำหรับของแต่ละชิ้น

ตัวอย่าง JavaScript — อ่านแล้วเข้าใจเลย
// 📦 Variable — เก็บชื่อ let name = "สมชาย"; let score = 85; // 🔧 Function — ตรวจผลสอบ function checkResult(score) { if (score >= 50) { return "✅ ผ่าน!"; } else { return "❌ ไม่ผ่าน"; } } // 👆 เมื่อกดปุ่ม → เรียก function button.addEventListener("click", () => { alert(checkResult(score)); // ผลลัพธ์: ✅ ผ่าน! });
⚛️

React

Library JavaScript ที่ทำให้สร้างแอพซับซ้อนได้ง่ายขึ้น

React คือ "เครื่องมือพิเศษ" ที่สร้างมาจาก JavaScript อีกทีหนึ่ง ทำให้สร้างแอพขนาดใหญ่ที่มีการโต้ตอบซับซ้อนได้โดยไม่งง

🧱
เปรียบเหมือน: ตัวต่อ LEGO

JavaScript ปกติเหมือนดินน้ำมัน — ปั้นได้ทุกอย่างแต่ยุ่งยาก
React เหมือน LEGO — มีชิ้นส่วนสำเร็จรูป (Component) ต่อเข้าหากันได้
สร้าง "ปุ่ม" ครั้งเดียว แล้วใช้ซ้ำได้ 100 ที่ — เปลี่ยนที่เดียวเปลี่ยนทุกที่

React vs JavaScript ธรรมดา ต่างกันอย่างไร?
JavaScript ธรรมดาReact
เปรียบเหมือน 🧱 ก่ออิฐทีละก้อน 🏗️ ใช้แผงสำเร็จรูป
ความยุ่งยาก ต้องจัดการ DOM เอง React จัดการ DOM ให้อัตโนมัติ
การใช้ซ้ำ Copy-paste โค้ด สร้าง Component ใช้ซ้ำได้
แอพขนาดใหญ่ จัดการยาก โค้ดพันกัน แยกส่วนชัดเจน จัดการง่าย
เหมาะกับ แอพเล็ก ง่ายๆ แอพกลาง-ใหญ่ ซับซ้อน
🃏
Component คืออะไร? เหมือนไพ่หนึ่งใบ

ลองนึกถึงเกมไพ่ แต่ละ "ใบไพ่" มีหน้าตาและการทำงานของตัวเอง
React Component ก็เหมือนกัน — <ProductCard /> คือ "ใบไพ่สินค้า" ที่สร้างครั้งเดียวแล้ววางได้ทุกที่
อยากแสดงสินค้า 100 ชิ้น ก็แค่ใช้ <ProductCard /> 100 ครั้ง

Backend (Server-side)

ส่วนที่ทำงานอยู่ "หลังม่าน" — ผู้ใช้มองไม่เห็น

Backend คือโค้ดที่รันอยู่บน Server ทำหน้าที่ ประมวลผลข้อมูล, จัดการ database, ตรวจสอบสิทธิ์ ก่อนส่งผลลัพธ์กลับมาให้ Frontend แสดง

🍳
เปรียบเหมือน: ครัวของร้านอาหาร

Frontend = หน้าร้าน (โต๊ะ เก้าอี้ เมนู) — ลูกค้ามองเห็น
Backend = ครัว — ลูกค้ามองไม่เห็น แต่ทำอาหารอยู่
ลูกค้าสั่งอาหาร (Frontend ส่ง request) → พ่อครัวทำอาหาร (Backend ประมวลผล) → เสิร์ฟมา (ส่งข้อมูลกลับ)

Database (ฐานข้อมูล)

คลังเก็บข้อมูลขนาดใหญ่ที่ค้นหาได้รวดเร็ว

Database คือที่เก็บข้อมูลแบบ "มีระบบ" — เก็บข้อมูลผู้ใช้ โพสต์ สินค้า ออเดอร์ ได้เป็นล้านๆ รายการ และค้นหาได้ทันที

📚
เปรียบเหมือน: ห้องสมุดที่มีบรรณารักษ์

ลองจินตนาการห้องสมุดที่มีหนังสือ 1 ล้านเล่ม — ถ้าหนังสือวางแบบสุ่ม หาหนังสือทีหนึ่งต้องใช้เวลาเป็นปี
แต่ถ้ามี บรรณารักษ์ จัดหมวดหมู่ไว้ เมื่อขอ "หนังสือผู้แต่งชื่อสมชาย" ได้ทันที
Database คือ "บรรณารักษ์" ที่จัดการข้อมูลให้มีระเบียบ

ประเภท Database หลัก
ประเภทเปรียบเหมือนตัวอย่างเหมาะกับ
SQL (ตาราง) 📊 สเปรดชีต Excel PostgreSQL, MySQL, SQLite ข้อมูลมีความสัมพันธ์กัน เช่น ร้านค้า, สั่งซื้อ
NoSQL (เอกสาร) 📁 โฟลเดอร์ JSON MongoDB, Firebase ข้อมูลยืดหยุ่น เช่น โพสต์ โซเชียล, โปรไฟล์ผู้ใช้
localStorage 📒 สมุดโน้ตในเบราว์เซอร์ ในตัวเบราว์เซอร์ แอพส่วนตัว ไม่ต้องแชร์กับใคร
SQL ทำงานอย่างไร?
🗣️
SQL = ภาษาสั่งให้ Database ทำงาน

SQL คือภาษาที่ใช้คุยกับ Database มันอ่านเหมือนภาษาอังกฤษปกติ:

SQL — อ่านออกเสียงได้เลย
-- "ดึงข้อมูล ทุกอย่าง จาก ตาราง users" SELECT * FROM users; -- "ดึงข้อมูล ชื่อ อีเมล จาก users ที่ อายุ มากกว่า 18" SELECT name, email FROM users WHERE age > 18; -- "เพิ่มข้อมูลใหม่ เข้าไป ใน users" INSERT INTO users (name, email) VALUES ("สมชาย", "som@email.com"); -- "ลบ จาก users ที่ ไม่มีการใช้งาน" DELETE FROM users WHERE active = false;

API

Application Programming Interface — ตัวเชื่อมต่อระหว่างแอพ

API คือ "ช่องทาง" ที่แอพหนึ่งใช้คุยกับแอพอื่น — เช่น แอพของคุณขอข้อมูลอากาศจาก Weather API หรือโพสต์รูปขึ้น Instagram ผ่าน Instagram API

🍽️
เปรียบเหมือน: พนักงานเสิร์ฟ

คุณ (แอพ) นั่งที่โต๊ะ → บอก พนักงานเสิร์ฟ (API) ว่าต้องการอะไร → พนักงานไปบอก ครัว (Database/Server) → ครัวทำอาหาร → พนักงานนำมาเสิร์ฟ

คุณไม่ได้เข้าไปในครัวเอง — คุณสื่อสารผ่าน API

API ในชีวิตจริง — คุณใช้อยู่ทุกวัน!

JSON คืออะไร? — กล่องส่งของมาตรฐาน

เมื่อ API ส่งข้อมูลกลับมา มักส่งในรูป JSON — เหมือนกล่องพัสดุมาตรฐานที่ทุกแอพ "แกะกล่อง" เป็น:
{ "name": "สมชาย", "age": 25, "city": "Bangkok" }

Hosting & Deploy

ทำให้คนทั้งโลกเข้าแอพของคุณได้

Deploy คือการ "เอาแอพขึ้น server" เพื่อให้ใครก็ได้เข้าถึงได้จาก URL — แทนที่จะรันได้แค่บนเครื่องของคุณ

🏠
เปรียบเหมือน: เปิดร้านอาหาร

ถ้าคุณทำอาหารอร่อยมาก แต่ทำกินแค่ที่บ้าน ไม่มีใครรู้
Deploy = เช่าพื้นที่เปิดร้าน ที่คนอื่นมากินได้
Hosting = เจ้าของอาคารที่ให้เช่าพื้นที่ (Netlify, Vercel, ฯลฯ)
Domain = ป้ายชื่อร้าน (myapp.com)

คำศัพท์ที่ต้องรู้
🌐
Domain (โดเมน) = ที่อยู่ของแอพ

เหมือนเลขบ้าน เช่น myshop.com หรือ myapp.netlify.app — ที่คนพิมพ์เพื่อเข้าแอพคุณ ซื้อจาก GoDaddy, Namecheap ราคาประมาณ 200-500 บาท/ปี

🏢
Hosting (โฮสติ้ง) = คอมพิวเตอร์ที่เก็บไฟล์แอพ

เหมือนพื้นที่เช่า — เอาไฟล์แอพไปไว้ที่นั่น Server ของ Hosting จะรันแอพตลอด 24 ชั่วโมง ผู้ใช้เข้าได้ทุกเมื่อ

🔒
HTTPS = ระบบป้องกัน (ต้องมี!)

เหมือนซองจดหมายปิดสนิท ข้อมูลที่รับส่งระหว่างผู้ใช้กับแอพถูกเข้ารหัส ป้องกันคนกลางดักอ่าน — Netlify/Vercel/GitHub Pages ให้ฟรีโดยอัตโนมัติ

เปรียบเทียบ Platform ฟรีสำหรับเริ่มต้น
Platformเหมาะกับความยากราคา
Netlify HTML/React ไฟล์เดียว ⭐ ลากวางได้เลย ฟรี
Vercel Next.js, React ⭐⭐ เชื่อม GitHub ฟรี
GitHub Pages HTML static ⭐⭐ ต้องมี GitHub ฟรี
Streamlit Cloud Python Streamlit ⭐⭐ เชื่อม GitHub ฟรี
Render Python/Node Backend ⭐⭐⭐ ต้องตั้งค่า ฟรี (จำกัด)
🎯
เริ่มต้นแนะนำ: Netlify

ไม่ต้องสมัคร GitHub ไม่ต้องรู้ command line — แค่ไปที่ netlify.com สมัคร account แล้ว drag-and-drop โฟลเดอร์ที่มีไฟล์ index.html ลงไป ได้ URL ทันทีใน 30 วินาที
